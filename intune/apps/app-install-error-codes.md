# <a name="intune-app-installation-error-reference"></a>Referência de erros de instalação de aplicativos do Intune

Além de seguir as etapas fornecidas para solucionar problemas de erros de aplicativo, você também pode aprender sobre erros de aplicativo específicos com base nos códigos de erro retornados. Depois de ter a correspondência de um código de erro, use a descrição e as informações adicionais para ajudar a resolver o erro.

Esta seção menciona o registro do DA (Administrador de dispositivo) e do Samsung Knox. Para obter mais informações, confira [Registro de administrador de dispositivos Android](../enrollment/android-enroll-device-administrator.md) e [Registrar automaticamente dispositivos Android usando o Knox Mobile Enrollment da Samsung](../enrollment/android-samsung-knox-mobile-enroll.md).

| 0xC7D14FB5 | -942583883 | Falha ao instalar o aplicativo.  | Essa mensagem de erro é exibida quando o Intune não consegue determinar a causa do erro de instalação do aplicativo do Android. Não foi fornecida nenhuma informação pelo Android durante a falha. Esse erro é retornado quando o download do APK é bem-sucedido, mas a instalação do aplicativo falha. Esse erro pode ocorrer com mais frequência devido a um arquivo APK inválido que não pode ser instalado no dispositivo. Uma possível causa pode ser quando o Google Play Protect bloqueia a instalação do aplicativo devido a questões de segurança.   Outra possível causa desse erro é quando um dispositivo não é compatível com o aplicativo. Por exemplo, se o aplicativo exigir a versão 21 ou superior da API e o dispositivo tiver a versão 19 da API no momento. O Intune retorna este erro para dispositivos DA e KNOX e, embora possa haver uma notificação de que os usuários podem clicar para tentar novamente, se houver um problema com o APK, ele nunca continuará a falhar.
